# PRD — 아이스크림 프로 (아이스크림연수원 B2G 홈페이지)

> 한 장짜리 제품 요구사항 문서 (v1).
> 벤치마크: 티처빌 프로 — https://www.teacherville.co.kr/b2g/support/index.edu
> 현재 구현체(`03.homepage`)를 기준으로 정리.

---

## 1. 한 줄 목적
교육청·교육지원청·재단 등 **기관 담당자**가 아이스크림연수원의 연수 과정을 둘러보고
**위탁연수를 빠르게 문의(견적요청)** 하도록 돕는 **B2G 영업 랜딩 사이트**.

## 2. 사용자
- **발주처 담당자**(교육청·교육지원청·재단 등 기관). 로그인 없이 열람·문의.
- (운영자) 접수된 문의는 추후 관리자 화면에서 확인 — 본 버전 범위 밖.

## 3. 핵심 기능
| 구분 | 내용 |
|---|---|
| 히어로 배너 | 브랜드 메시지 + 슬라이드(자동 전환) |
| 연수 과정 목록 | 카드 그리드(분류·학점·가격·상태), 과정 상세 링크 |
| 분류(분야)별 필터 | 디지털역량·인문소양·교수학습·어학·학급경영 등으로 좁히기 |
| 담당분야별 탐색 | 분야 아이콘으로 과정 찾기 |
| 빠른 문의(견적요청) | 기관명·지역·연락처·문의내용 입력 → 접수 |
| 도입 절차 안내 | 문의→제안→계약→운영→정산 단계 |

## 4. 데이터 모델 (요약)
Supabase 테이블 3종 (상세 DDL: `supabase/schema.sql`):
- **courses** — id, title, credit, category, category_name, price, status, url
- **categories** — key, label, icon, sort
- **inquiries** — id, org, region, phone, message, status, created_at

## 5. 화면 구성 (대략)
단일 랜딩 페이지:
```
헤더/GNB (로고·메뉴·로그인/회원가입)
  └ 히어로 배너 (메시지 + 슬라이드)
  └ [좌] 연수 과정 목록 + 분류 필터   |   [우] 빠른 문의 폼
  └ 담당분야별 탐색 (아이콘 그리드)
  └ 도입 절차 (단계 카드)
  └ 푸터 (회사정보 · 고객센터 1544-3070)
```

## 6. 저장 방식
- **Supabase(PostgreSQL)** 를 DB로 사용.
  - 과정·분류: **DB에서 읽기**(공개 읽기).
  - 문의: **DB에 저장(insert)**.
- 프론트는 `src/data/api.js` **단일 창구**로 Supabase REST API 호출
  (anon key + RLS로 *공개 읽기*와 *문의 등록*만 허용, 그 외 차단).
- 빌드 없는 정적 사이트에서 서버 코드 없이 직접 연결.

## 7. 성공 기준
- [ ] 기관 담당자가 별도 설명 없이 **과정 탐색·분류 필터·문의**를 할 수 있다.
- [ ] 제출한 문의가 **DB에 적재**되어 운영자가 나중에 확인할 수 있다.
- [ ] 과정/분류가 **DB 내용과 일치**해 표시된다.

## 8. 비범위 (Out of Scope)
- 결제, 수강·이수 관리(LMS) 기능
- 회원가입·로그인·권한
- 관리자 화면(문의·과정 관리) — 다음 단계
- 외부 연수원과의 실시간 자동 동기화(CORS·봇차단으로 불가)
